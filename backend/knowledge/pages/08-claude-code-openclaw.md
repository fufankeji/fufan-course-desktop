# Claude Code 与 OpenClaw 自动化研发

这一页是 Vibe Coding 课程并入 Agent 大课后的连接点。它强调不是“让 AI 写几行代码”，而是建立可持续的工程协作流程。

## 工作流

- 需求澄清：让 Agent 复述目标和验收标准。
- 任务拆分：把需求拆成可测试的小步。
- 测试先行：先写失败测试，再实现功能。
- 运行验证：用测试、构建和截图证明效果。
- 交付总结：说明改动、验证结果和后续风险。

## 适合学员练习

学员可以让 Agent 基于课程 Wiki 开发一个微型工具，例如课程问答系统、评估集生成器或 Skill 安装器。

## 常见误区

只看生成速度不看验证，会让项目变成一次性 Demo。课程要求每个 Vibe Coding 项目都配测试命令。
